# Alto Cloud 機能紹介

**Alto Cloud はローカルネットワークを優先するファイル共有アプリです。2 台間では Quick Send、複数端末では Shared Cloud を使用します。**

アカウントやインターネット上のクラウドを必要とせず、家庭、会議、教室、スタジオ、撮影現場などで近くの端末同士がファイルを交換できます。

## 製品モデル

Alto Cloud = **Quick Send + 任意の Shared Cloud セッション**。

- **Quick Send**：近くの端末への 1 対 1 送信
- **Shared Cloud**：1 台の端末がホストする一時的な共有スペース
- **One-Time Drop**：指定した全端末が開いた後に自動削除されるファイル

## 主な機能

| 機能 | 動作 |
| --- | --- |
| 同一 Wi-Fi 検出 | Bonjour/mDNS で Alto Cloud 端末を検出します。 |
| Quick Send | 各端末を直接送信先として公開します。 |
| Shared Cloud | 1 台の端末をグループ用ローカルサーバーにします。 |
| 二重の役割 | Shared Cloud ホストも Quick Send の送信先として残ります。 |
| 保存容量制限 | Shared Cloud が使用できるローカル容量を制限します。 |
| ローカル保存先 | ホストが共有ファイルの保存フォルダを選択できます。 |
| One-Time Drop | 対象端末を限定し、全対象が開いた後に削除します。 |

## One-Time Drop のルール

送信者が対象端末を選択します。後から参加した端末には自動的に表示されません。指定したすべての端末が開いた後、Alto Cloud がファイルを削除します。

これは保存期間のルールであり DRM ではありません。スクリーンショット、外部撮影、閲覧後のコピーを防止するものではありません。

## LAN 基盤

- サービス種別：`_altocloud._tcp`
- 常時公開される Quick Send peer
- Shared Cloud 開始時に追加される共有ノード
- 1 台の Mac が Quick Send の送信先と Shared Cloud ホストの両方として表示可能

現在のプロトタイプは LAN 検出とファイル状態のシミュレーションを実装しています。実際の制御チャネルとファイルストリーミングは次の開発段階です。
